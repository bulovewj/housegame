## 1. 프로젝트 한 줄 요약

부동산금융 개념(LTV·대출·이자·시세차익·월세)을 **타이쿤 시뮬레이션**으로
체험 학습하는 **모바일 세로 웹앱**. 도트+부드러운 UI 혼합, 무한 모드.
대상: 초등 고학년 ~ 중1 (6~7학년).

## 2. 핵심 원칙 (개발 내내 지킬 것)

1. **게임 반 · 학습 반** — 재미와 교육이 5:5. 학습이 게임을 방해하지 않되, 개념이 메카닉에 실제로 녹아 있어야 함.
2. **현실 기반 설계** — LTV 등은 은행 재량이 아니라 **정부 규제**로 표현. 수치는 현실을 단순화하되 왜곡하지 않음.
3. **연산은 단순하게** — 학생이 암산 가능한 수준(×0.7, 덧셈·뺄셈, 반올림)으로. 복잡한 복리·소수점은 내부 처리하고 화면엔 반올림 표기.
4. **변동성은 예측 가능하게** — 랜덤 요소는 범위를 명시하고 튜토리얼에서 설명. 학생이 "왜 이렇게 됐는지" 이해할 수 있어야 함.
5. **개념 설명은 명시적으로** — 툴팁/미니게임/학습카드로 LTV 등을 직접 노출. 원하면 더 볼 수 있게, 강요하지 않게.
6. **모바일 세로 우선** — 모든 UI는 세로 화면(약 390×844) 기준으로 먼저 설계.


## 6. 부동산금융 메카닉 (학습 개념 → 게임 규칙)

| 개념 | 게임에서의 구현 | 학습 노출 방식 |
|---|---|---|
| **LTV** | 집값의 일정 %까지만 대출 가능 (예: 70%). 정부 규제값으로 고정, 이벤트로 조정 | 구매 시 "집값 × 0.7 = 대출한도" 툴팁 |
| **대출·이자** | 대출 실행 시 원금 발생, 매턴 이자 차감 | 대출 확인 화면에 원금·이자 분리 표시 |
| **기준금리 변동** | 매턴 또는 이벤트로 금리 변동 → 이자 상환액 변동 | 금리 변동 시 알림 + "금리가 오르면 이자가 늘어요" |
| **집값 변동** | 입지·컨디션별로 다른 변동폭. 좋은 입지=변동 큼/상승 경향 등 | 건물 정보에 입지·컨디션 등급 표시 |
| **시세차익** | 싸게 사서 오른 뒤 팔면 차익. 판매 시 손익 표기 | 판매 결과에 "구매가 vs 판매가" 비교 |
| **월세/임대수익** | 보유 건물에서 매턴 월세 입금 | 건물별 예상 월세 표시, 턴 정산에 합산 |

> **미확정(⚠️)**: LTV 비율, 금리 범위, 변동폭, 월세 수익률 등 구체 수치는
> §9 밸런싱에서 초안 → 상의 후 확정.

## 9. 밸런싱 (2026-07-03 확정 · `src/game/balance.js`가 단일 출처)

| 파라미터 | 확정값 | 메모 |
|---|---|---|
| 시작 자본 | 5,000만 | 첫 집을 대출 끼고 살 수 있는 수준 |
| LTV | 70% | 정부 규제값, ×0.7 암산 용이 |
| 기준금리 초기값 | 연 5% | 최대 대출 시 이자≈월세가 되어 금리 이벤트가 실질 위협이 되도록 |
| 금리 변동폭 | ±0.5%p/이벤트, 범위 1~10% | 3턴마다 경제 상황 이벤트 예고 → 다음 턴 반영 |
| 집값 변동 | low -1~+1.5% / medium -3~+4% / high -6~+7.5% /턴 | 등급이 높을수록 기대치·위험 모두 커짐 |
| 월세 수익률 | 집값의 0.3%/턴 | 정산 시 현금 입금 |
| 수리·관리비 | 2턴마다 컨디션별 집값의 0.1%(좋음)/0.2%(보통)/0.4%(낮음) | 월세 주는 집의 유지비용. 낡을수록 비쌈 |
| 컨디션 가격 보정 | 컨디션 '낮음' 건물은 매턴 -0.5%p 드리프트 | 낡은 건물은 기대치가 마이너스 → 실제로 하락(30턴 하락확률 96%) |
| 취득세 | 구매가의 2% | 거래비용 학습 + 잦은 매매 억제 |
| 미니게임 보상 | 1회 50~200만 랜덤 | 부동산 수익이 주인공이 되도록 축소. 하루 1회 제한 |
| 매물 시장가 | 미보유 매물도 매턴 변동 | 판매한 건물은 판매가로 시장 복귀 (재구매 차익 방지) |

> 근거: 30턴 시뮬레이션에서 "일하기만" 0.88억 vs "고위험 매물+최대대출" 평균 1.23억·손실확률 2~9%.
> 리스크-리턴 구조와 대출 레버리지 학습이 성립함을 확인.
