## 1. 프로젝트 한 줄 요약

부동산금융 개념(LTV·대출·이자·시세차익·월세)을 **타이쿤 시뮬레이션**으로
체험 학습하는 **모바일 세로 웹앱**. 도트+부드러운 UI 혼합, 무한 모드.
대상: 초등 고학년 ~ 중1 (6~7학년).

## 2. 핵심 원칙 (개발 내내 지킬 것)

1. **게임 반 · 학습 반** — 재미와 교육이 5:5. 학습이 게임을 방해하지 않되, 개념이 메카닉에 실제로 녹아 있어야 함.
2. **현실 기반 설계** — LTV 등은 은행 재량이 아니라 **정부 규제**로 표현. 수치는 현실을 단순화하되 왜곡하지 않음.
3. **연산은 단순하게** — 학생이 암산 가능한 수준(×0.7, 덧셈·뺄셈, 반올림)으로. 복잡한 복리·소수점은 내부 처리하고 화면엔 반올림 표기.
4. **변동성은 예측 가능하게** — 랜덤 요소는 범위를 명시하고 튜토리얼에서 설명. 학생이 "왜 이렇게 됐는지" 이해할 수 있어야 함.
5. **개념 설명은 명시적으로** — 툴팁/미니게임/학습카드로 LTV 등을 직접 노출. 원하면 더 볼 수 있게, 강요하지 않게.
6. **모바일 세로 우선** — 모든 UI는 세로 화면(약 390×844) 기준으로 먼저 설계.

## 4. 게임 정체성

- **장르**: 부동산 타이쿤 시뮬레이션 (싱글플레이)
- **모드**: 무한 모드 (엔딩 없음, 자산을 계속 불려감)
- **목표**: 대출·시세차익·월세로 현금을 불려 집을 계속 늘려가며 순자산 증식
- **레퍼런스**: 무한의 게임(도트 감성), 비즈앤타운(타이쿤 구조)
- **화면 방향**: 모바일 세로

## 5. 핵심 게임 루프

```
[기본 화면 = 도트 맵]  내 건물들이 맵 위에 보임
        │
        ├─ 맵의 건물 탭 → 컨디션·시세·월세 정보 확인
        │
        └─ 하단 메뉴 선택
             ├─ 🏠 부동산 구매   (매물 카드 열람 → LTV 대출 활용 구매)
             ├─ 💰 부동산 판매   (보유 건물 시세로 매각, 시세차익 실현)
             ├─ 🔨 일하기        (랜덤 미니게임 → 현금 획득)
             └─ 🏦 대출 확인      (대출 잔액·현재 금리·이번 턴 이자 확인)
        │
   [한 턴 소비 = 다음날]
        │
        ▼
  턴 종료 정산: 대출 이자 차감 · 집값 변동(입지·컨디션별) · 월세 수입 입금 · 이벤트 발생 가능
```

## 6. 부동산금융 메카닉 (학습 개념 → 게임 규칙)

| 개념 | 게임에서의 구현 | 학습 노출 방식 |
|---|---|---|
| **LTV** | 집값의 일정 %까지만 대출 가능 (예: 70%). 정부 규제값으로 고정, 이벤트로 조정 | 구매 시 "집값 × 0.7 = 대출한도" 툴팁 |
| **대출·이자** | 대출 실행 시 원금 발생, 매턴 이자 차감 | 대출 확인 화면에 원금·이자 분리 표시 |
| **기준금리 변동** | 매턴 또는 이벤트로 금리 변동 → 이자 상환액 변동 | 금리 변동 시 알림 + "금리가 오르면 이자가 늘어요" |
| **집값 변동** | 입지·컨디션별로 다른 변동폭. 좋은 입지=변동 큼/상승 경향 등 | 건물 정보에 입지·컨디션 등급 표시 |
| **시세차익** | 싸게 사서 오른 뒤 팔면 차익. 판매 시 손익 표기 | 판매 결과에 "구매가 vs 판매가" 비교 |
| **월세/임대수익** | 보유 건물에서 매턴 월세 입금 | 건물별 예상 월세 표시, 턴 정산에 합산 |

> **미확정(⚠️)**: LTV 비율, 금리 범위, 변동폭, 월세 수익률 등 구체 수치는
> §9 밸런싱에서 초안 → 상의 후 확정.

## 8. 프로젝트 구조 (초안 ⚠️)

```
src/
  main.jsx
  App.jsx
  game/
    state.js         # 게임 상태 정의(현금·턴·금리·보유건물·대출)
    engine.js        # 턴 처리, 정산 로직
    balance.js       # 밸런싱 상수 (LTV·금리범위·변동폭 등) ← 한곳에 모음
    events.js        # 이벤트 카드 정의
  data/
    properties.js    # 매물 카드 데이터
    concepts.js      # 학습 개념 텍스트(PDF 기반)
  minigames/         # 랜덤 미니게임 모음
  components/
    Map/             # 도트 맵
    Menu/            # 하단 메뉴
    Panels/          # 구매·판매·대출·정보 패널
    Tooltip/         # 학습 툴팁
  ui/                # 공통 도트+부드러운 UI 컴포넌트
  save/
    storage.js       # localStorage 저장/불러오기
  assets/            # 스프라이트·아이콘
```

## 9. 밸런싱 초안 (⚠️ 전부 미확정 · 상의 후 확정)

| 파라미터 | 초안값 | 메모 |
|---|---|---|
| 시작 자본 | 5,000만 (게임 단위 조정 가능) | 첫 집을 대출 끼고 살 수 있는 수준 |
| LTV | 70% | 정부 규제값, ×0.7 암산 용이 |
| 기준금리 초기값 | 연 4% | 턴당 이자 = 대출잔액 × (금리/12) 단순화 |
| 금리 변동폭 | ±0.5%p/이벤트 | 범위 명시 |
| 집값 변동 | 입지별 -3%~+5%/턴 | 컨디션이 낮으면 상승폭 작음 |
| 월세 수익률 | 집값의 0.3%/턴 | 정산 시 현금 입금 |
| 미니게임 보상 | 1회 100~500만 랜덤 | 종류별 편차 |

> 위 숫자는 **자리만 잡아둔 값**. 실제 밸런스 테스트로 조정 필요.
