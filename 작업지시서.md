## 1. 만들 것 (What)

부동산금융을 배우는 **모바일 세로 타이쿤 웹앱**.
- 도트 맵에서 건물을 사고팔며 자산을 불림
- LTV 대출·기준금리 이자·시세차익·월세 수익이 핵심 메카닉
- 무한 모드, 게임 반·학습 반
- React + Vite, localStorage 수동 저장

---

## 2. 절대 규칙 (Must)

1. **모바일 세로 기준**으로 먼저 만든다 (약 390×844). 데스크톱은 나중.
2. **연산은 학생이 암산 가능**해야 한다. 화면 숫자는 반올림. 복잡한 계산은 내부에서만.
3. **밸런싱 상수는 한 파일(`balance.js`)에 모은다.** 코드 여기저기 흩지 말 것.
4. **학습 개념 텍스트는 `data/concepts.js`에 모은다.** PDF 기반, 초등·중1 눈높이.
5. **랜덤은 항상 범위가 명확**해야 하고 튜토리얼/툴팁으로 설명 가능해야 한다.
6. **저장은 수동.** 자동저장·서버 없음. localStorage에 저장/불러오기 버튼.
7. **커밋은 작게, 자주.** 마일스톤 태스크 단위로.

- 확정 안 된 수치를 "확정"처럼 쓰지 말 것 — 모두 `balance.js`에서 조정 가능하게.

## 4. 화면 명세 (세로 모바일)

### 4.1 기본 화면: 맵
- 상단 HUD: 현금 / 순자산 / 현재 턴(일차) / 기준금리
- 중앙: 도트 맵. 내 건물이 배치됨. 건물 탭 → 정보 패널.
- 하단 메뉴 바: [부동산 구매] [부동산 판매] [일하기] [대출 확인]
- 우상단: [저장] [불러오기] [다음날(턴 종료)]

### 4.2 부동산 구매 패널
- 매물 카드 리스트(입지·컨디션·가격·예상 월세)
- 카드 선택 → 상세: LTV 대출한도 자동 계산 표기 ("집값 × 0.7 = 대출한도")
- [현금 구매] / [대출 끼고 구매] 버튼
- 대출 시 발생 원금·예상 턴당 이자 미리보기

### 4.3 부동산 판매 패널
- 보유 건물 리스트, 현재 시세
- 선택 → "구매가 vs 현재 시세" 손익 표기
- [판매] → 시세차익/손실 반영

### 4.4 일하기 패널
- 랜덤으로 미니게임 1종 실행
- 결과에 따라 현금 획득 → 턴 소비 여부는 설계 결정(⚠️)

### 4.5 대출 확인 패널
- 대출 원금 잔액 / 현재 기준금리 / 이번 턴 이자 / 상환 버튼
- "금리가 오르면 이자가 늘어요" 학습 문구

### 4.6 턴 종료(다음날) 정산 화면
- 이자 차감 / 집값 변동 / 월세 입금 / 이벤트를 한 화면에 요약

## 5. 데이터 규약

### 5.1 게임 상태 (state.js)
```js
{
  day: 1,
  cash: 50_000_000,
  interestRate: 0.04,      // 기준금리
  properties: [ /* 보유 건물 */ ],
  loan: { principal: 0 },  // 대출 원금
}
```

### 5.2 매물 카드 (data/properties.js)
```js
{
  id, name, location,       // 입지 등급
  condition,                // 컨디션 등급
  price, expectedRent,      // 시세, 예상 월세
  priceVolatility,          // 입지·컨디션 기반 변동 특성
  conceptTag                // 연결된 학습 개념(예: 'LTV')
}
```

### 5.3 학습 개념 (data/concepts.js)
```js
{
  key: 'LTV',
  title: '집값의 몇 %까지 빌릴 수 있을까?',
  kidText: '...(초등·중1 눈높이 설명)...',
  source: '부동산_압축.pdf Ch.9'
}
```

## 6. 계산 규칙 (단순화)

- **대출한도** = 집값 × LTV (기본 0.7)
- **턴당 이자** = 대출원금 × (기준금리 ÷ 12)  → 화면엔 반올림
- **집값 변동** = 현재가 × (1 + 입지·컨디션별 변동률)
- **월세 수입** = 집값 × 월세율(기본 0.003)
- 모든 표시 금액은 만원 단위 반올림 권장

> 상수(LTV, 금리, 변동률, 월세율)는 전부 `balance.js`에서 관리.
