## 1. 화면 기본 규격

- 세로 모바일 전용, 기준 해상도 **390 × 844px** (`app-viewport`, `src/App.css`)
- 화면이 이 크기보다 작은 기기에서도 `max-width: 100vw`로 가로만 줄어들고, 세로는 최소 844px 유지
- 폰트
  - 픽셀 느낌: `--font-pixel: 'Courier New', ui-monospace, monospace` → 상단 타이틀에만 사용
  - 본문: `--font-body: system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif` → 나머지 전부
- 다크 테마 고정(`color-scheme: dark`)

## 2. 전체 화면 구조 (세로 스택)

```
┌─────────────────────────────────┐
│ 상단바(topbar)                   │  타이틀 "부동산 타이쿤" + 📖/저장/불러오기/다음날 버튼
├─────────────────────────────────┤
│ HUD (4칸 그리드)                 │  현금 · 순자산 · 일차 · 기준금리
├─────────────────────────────────┤
│ 시즌 목표 바 (접이식)             │  시즌 N · d/10일 · ⭐개수 (펼치면 목표 체크리스트 3개)
├─────────────────────────────────┤
│                                 │
│         맵 (game-map)           │  체크보드 초록 배경 + 보유 건물 버튼들 (2열 그리드)
│         flex: 1 (남는 공간 전부)  │
│                                 │
├─────────────────────────────────┤
│ 하단 메뉴바 (4칸 그리드)          │  부동산 구매 · 부동산 판매 · 일하기 · 대출 확인
└─────────────────────────────────┘

  (패널/모달은 위 전체를 덮는 바텀시트로 뜸: panel-overlay → panel-sheet)
```

패널은 전부 **화면 하단에서 위로 슬라이드 올라오는 바텀시트** 형태(`max-height: 85%`, 넘치면 내부 스크롤)입니다. 반투명 검은 오버레이(`rgba(0,0,0,0.5)`) 위에 뜹니다.

## 3. 버튼 레이아웃 상세

### 3.1 상단바 (`.app-topbar`)
왼쪽부터: 타이틀(고정 텍스트) — 오른쪽 정렬 버튼 그룹(줄바꿈 가능)

| 버튼 | 라벨/아이콘 | 동작 | 스타일 |
|---|---|---|---|
| 학습노트 | 📖 (아이콘만, `aria-label="학습 노트"`) | 학습 개념 카드 목록 열기 | 기본(surface-soft) |
| 저장 | "저장" | localStorage에 현재 상태 저장 | 기본 |
| 불러오기 | "불러오기" | localStorage에서 불러오기 | 기본 |
| 다음날 | "다음날" | 턴 진행(정산) | 강조색(accent) 테두리/텍스트 |

버튼 높이 `min-height: 36px`, 폰트 11px — 4개가 한 줄에 들어가되 좁으면 줄바꿈.

### 3.2 HUD (`.hud`) — 4칸 그리드, 버튼 아님(정보 표시 전용)
현금 / 순자산 / 일차 / 기준금리 — 각각 라벨(작게, dim) + 값(굵게) 세로 배치, 카드형 배경.

### 3.3 시즌 목표 바 (`SeasonGoals`)
`<details>` 네이티브 접이식. 접었을 땐 "시즌 1 · 3/10일 · ⭐ 0" 한 줄만 보이고, 펼치면 목표 3개 리스트(달성 시 취소선+강조색):
- 순자산 1억원
- 임대수익 1,000만원
- 구제금 없이 생존

### 3.4 맵 (`Map.jsx`)
- 배경: CSS로 그린 초록 체크보드 패턴(진짜 이미지 아님, `linear-gradient` 2개 겹침, 56px 격자)
- 건물 없을 때: 가운데 안내 문구 "첫 건물을 구매해 도시를 시작하세요!"
- 건물 있을 때: **2열 그리드**(`game-map-grid`)에 보유 건물 수만큼 버튼이 순서대로 채워짐 (5종 매물 → 최대 2열×3행)
- 건물 버튼(`.game-map-house`) 현재 모습: **72×72px 정사각형, 배경색 `--color-cash`(황금색), 테두리만 있는 색상 박스** — 아직 그림/스프라이트 없음. 버튼 아래에 건물 이름이 작은 텍스트(10px)로 표시됨
- 클릭 시 `PropertyDetailPanel`(건물 상세 패널) 오픈

> 🎨 **이미지 작업 포인트**: `.game-map-house`의 배경색 박스를 실제 건물 스프라이트로 교체하면 됩니다. 그리드 셀 크기는 유동적(`minmax(72px, 1fr)`)이라 정사각형 그림이면 대체로 무난하게 들어갑니다.

### 3.5 하단 메뉴바 (`MenuBar.jsx`) — 4칸 그리드
| 키 | 라벨 | 비활성 조건 |
|---|---|---|
| buy | 부동산 구매 | 없음(항상 활성) |
| sell | 부동산 판매 | 없음 |
| work | 일하기 | 오늘 이미 일했으면 메뉴에서 제외 |
| loan | 대출 확인 | 없음 |

버튼 높이 `min-height: 44px`(터치 영역 확보).

### 3.6 패널별 버튼 구성

| 패널 | 열리는 경로 | 주요 버튼 |
|---|---|---|
| **PurchasePanel** (부동산 구매) | 메뉴바 "부동산 구매" | 매물 목록 → 선택 → 대출 프리셋 3개(대출없이/절반만/최대로) + 슬라이더 → "대출 끼고 구매"/"현금으로 구매" |
| **SellPanel** (부동산 판매) | 메뉴바 "부동산 판매" / 상세패널 "매도하기" | 매물 목록 → 선택 → "판매" |
| **LoanPanel** (대출 확인) | 메뉴바 "대출 확인" / 상세패널 "상환하기" | 담보대출 select + 상환 슬라이더 + "가능한 만큼 전액" + "부분상환" |
| **WorkPanel** (일하기) | 메뉴바 "일하기" | 미니게임 4종 중 1개 랜덤 실행 |
| **RepairPanel** (건물 수리) | 상세패널 "수리하기" | 건물 select + "수리 시작" → 리듬 미니게임 → "완료" |
| **PropertyDetailPanel** (건물 상세) | 맵에서 건물 탭 | "🔨 수리하기" / "🏦 상환하기"(대출 없으면 비활성) / "💰 매도하기" |
| **ConceptBookPanel** (학습노트) | 상단바 📖 | "🦊 튜토리얼 다시보기" + 개념 카드 목록(펼치기 방식) |
| **TurnSummaryPanel** (턴 정산) | "다음날" 클릭 후 자동 표시 | 보통은 "확인"만, 선택형 이벤트가 뜨면 "투자"/"현금 보유" 버튼으로 대체(강제 선택) |

모든 패널 상단엔 공통 "✕" 닫기 버튼(`aria-label="닫기"`, 탭 영역 40×40px)이 있습니다.

## 4. 캐릭터

현재 캐릭터는 **여우 마스코트 🦊 이모지 하나뿐**이며, 별도 스프라이트/일러스트 없이 이모지를 그대로 CSS `font-size: 40px`로 키워서 씁니다.

- 등장 위치: 최초 진입 튜토리얼(`TutorialOverlay`)에서만 등장 (7단계 대사)
- 학습노트 패널의 "튜토리얼 다시보기" 버튼 라벨에도 🦊 이모지 재사용
- 게임 플레이 중(맵 화면 등)에는 등장하지 않음 — 상시 등장하는 안내 캐릭터로 확장할 여지 있음

> 🎨 **이미지 작업 포인트**: 여우 캐릭터를 실제 일러스트/스프라이트로 만들면 `TutorialOverlay.jsx`의 `<span className="tutorial-mascot">🦊</span>` 자리에 `<img>`로 교체하면 됩니다(현재 40px 폰트 크기 기준 배치).

## 5. 건물(매물) 데이터 — 전체 5종 (`src/data/properties.js`)

| id | 이름 | 입지 | 컨디션 | 가격 | 변동성 | 지역 특성(regionTrait) | 지역 드리프트 |
|---|---|---|---|---|---|---|---|
| `studio-old` | 낡은 원룸 | 변두리 | 낮음 (bad) | 3,000만원 | low | 저렴한 매입가 · 높은 관리 부담 | -0.001 |
| `multi-house` | 다세대주택 | 보통 | 보통 (normal) | 6,000만원 | medium | 안정적인 주거 수요 | 0 |
| `flawed-shophouse` | 하자있는 상가주택 | 좋음 | 낮음(하자) (bad) | 8,000만원 | high | 상권 호재 · 높은 변동성 | +0.001 |
| `new-villa` | 신축 빌라 | 좋음 | 좋음 (good) | 12,000만원 | medium | 신축 선호 · 안정적 관리 | +0.0005 |
| `station-apartment` | 역세권 구축 아파트 | 매우 좋음 | 보통 (normal) | 15,000만원 | high | 교통 수요 · 높은 성장 가능성 | +0.0015 |

- **컨디션 등급(good/normal/bad)** 은 관리비 요율·가격 드리프트·미니게임 수리 대상 판단에 쓰입니다(각 건물은 컨디션 점수 0~100을 별도로 가지며 등급은 점수에서 파생: 75↑ good, 45↑ normal, 그 외 bad)
- **변동성(volatility)** 은 턴마다 가격이 얼마나 출렁이는지 결정(low/medium/high 3단계)
- 5종 모두 `conceptTag: 'LTV'`를 갖고 있어 구매 화면에서 LTV 개념 툴팁이 뜸

> 🎨 **이미지 작업 포인트**: 매물 5종은 컨디션(낮음/보통/좋음)과 입지(변두리~매우 좋음) 조합이 서로 다르므로, 건물 그림도 이 정보가 시각적으로 구분되면 좋습니다 (예: 낡은 원룸=허름한 저층 건물, 역세권 아파트=고층 신식 건물). 현재는 5개 모두 동일한 노란 정사각형 박스로만 표시되어 구분이 안 됩니다.
