# 마일스톤별 개발 계획 — 부동산 타이쿤 웹앱

> 위에서 아래로 순서대로 진행합니다. 한 마일스톤을 끝내고 동작을 확인한 뒤
> 다음으로 넘어가세요. 각 마일스톤은 "이것만 되면 완료"인 명확한 목표가 있습니다.
> 원진님의 작업 방식(점진적 결정)에 맞춰, 밸런싱·미니게임 등은 뒤 마일스톤에서
> 상의하며 확정합니다.

---

## M0. 프로젝트 셋업  ⏱️ 가장 먼저

**목표**: React + Vite 앱이 모바일 세로 화면으로 뜬다.

- [ ] Vite + React 프로젝트 생성
- [ ] 모바일 세로 기준 레이아웃 골격(390×844 컨테이너, 중앙 정렬)
- [ ] 폴더 구조 잡기 (`CLAUDE.md §8` 따름)
- [ ] `balance.js`, `state.js` 빈 뼈대 생성
- [ ] 도트+부드러운 UI를 위한 기본 색/폰트/버튼 컴포넌트 1개

**완료 기준**: `npm run dev`로 세로 화면에 "부동산 타이쿤" 타이틀이 뜬다.

---

## M1. 맵 + HUD (기본 화면)

**목표**: 도트 맵과 상단 정보(현금·순자산·턴·금리)가 보인다.

- [ ] 도트 맵 렌더링 (임시 타일이라도)
- [ ] 내 건물 스프라이트를 맵에 배치(초기엔 0~1채)
- [ ] 상단 HUD: 현금 / 순자산 / 일차(day) / 기준금리
- [ ] 하단 메뉴 바 4버튼 배치 (구매·판매·일하기·대출) — 아직 동작 X
- [ ] 우상단 [다음날] 버튼 배치 — 아직 동작 X

**완료 기준**: 맵·HUD·메뉴가 세로 화면에 자연스럽게 배치된다.

---

## M2. 턴 시스템 (다음날 루프)

**목표**: [다음날]을 누르면 day가 1 증가하고 정산이 돈다.

- [ ] 턴 종료 → `day++`
- [ ] 턴 정산 파이프라인: (이자 → 집값 변동 → 월세 입금 → 이벤트) 순서 프레임
- [ ] 턴 종료 정산 요약 화면(변화 내역 표시)
- [ ] `engine.js`에 정산 로직 분리

**완료 기준**: 다음날을 누를 때마다 날짜가 넘어가고 요약이 뜬다(값은 임시라도).

---

## M3. 부동산 구매 + LTV 대출  ⭐ 학습 핵심 1

**목표**: 매물을 대출 끼고 살 수 있고, LTV 개념이 노출된다.

- [ ] `data/properties.js` 매물 카드 데이터(Claude 창작, PDF 개념 연결)
- [ ] 구매 패널: 매물 리스트 + 상세
- [ ] LTV 대출한도 자동 계산 표기("집값 × 0.7 = 대출한도")
- [ ] [현금 구매] / [대출 끼고 구매]
- [ ] 대출 실행 시 `loan.principal` 증가, 보유 건물에 추가
- [ ] LTV 학습 툴팁 (`concepts.js`)

**완료 기준**: 대출로 집을 사면 현금·대출·보유건물이 정확히 바뀐다.
**상의 포인트(⚠️)**: LTV 비율, 매물 가격표 확정.

---

## M4. 대출 이자 + 기준금리 변동  ⭐ 학습 핵심 2

**목표**: 매턴 이자가 나가고, 금리가 변하면 이자도 변한다.

- [ ] 턴 정산에 이자 차감 연결 (대출원금 × 금리/12)
- [ ] 기준금리 변동 로직(이벤트/주기) — 범위 명시
- [ ] 대출 확인 패널: 원금·금리·이번 턴 이자·상환 버튼
- [ ] 금리 변동 알림 + "금리 오르면 이자 늘어요" 문구
- [ ] 상환 기능(현금으로 원금 갚기)

**완료 기준**: 턴을 넘기면 이자가 빠지고, 금리 변동이 이자에 반영된다.
**상의 포인트(⚠️)**: 초기 금리, 변동폭, 변동 주기 확정.

---

## M5. 부동산 판매 + 시세차익  ⭐ 학습 핵심 3

**목표**: 보유 건물을 팔아 시세차익/손실을 실현한다.

- [ ] 집값 변동 로직(입지·컨디션별 변동폭) 턴 정산에 연결
- [ ] 판매 패널: 보유 건물 + 현재 시세
- [ ] "구매가 vs 현재 시세" 손익 표기
- [ ] 판매 시 현금 반영, 보유 목록에서 제거
- [ ] 시세차익 학습 툴팁

**완료 기준**: 오른 뒤 팔면 차익이, 내린 뒤 팔면 손실이 정확히 반영된다.
**상의 포인트(⚠️)**: 입지·컨디션별 변동폭 확정.

---

## M6. 월세/임대 수익

**목표**: 보유 건물에서 매턴 월세가 들어온다.

- [ ] 건물별 예상 월세 표시
- [ ] 턴 정산에 월세 입금 합산
- [ ] 월세 개념 학습 툴팁

**완료 기준**: 다음날마다 보유 건물 월세만큼 현금이 는다.
**상의 포인트(⚠️)**: 월세 수익률 확정.

---

## M7. 일하기 미니게임 (랜덤)

**목표**: 일하기를 누르면 랜덤 미니게임으로 현금을 번다.

- [ ] 미니게임 프레임(공통 인터페이스: 시작→결과→보상)
- [ ] 미니게임 2~3종 구현(예: 반사·간단퀴즈·미로 중 상의)
- [ ] 일부 미니게임에 금융개념 퀴즈 연결
- [ ] 보상 → 현금 반영, 턴 소비 여부 결정

**완료 기준**: 랜덤으로 미니게임이 뜨고 보상이 현금에 반영된다.
**상의 포인트(⚠️)**: 미니게임 종류·개수·보상액 확정.

---

## M8. 저장 / 불러오기 (수동)

**목표**: 버튼으로 진행 상황을 저장하고 이어서 할 수 있다.

- [ ] `save/storage.js` — localStorage 직렬화/역직렬화
- [ ] [저장] [불러오기] 버튼 동작
- [ ] 저장 슬롯(1개 또는 여러 개) 결정
- [ ] 불러오기 시 전체 상태 복원 검증

**완료 기준**: 저장 후 새로고침 → 불러오기 하면 상태가 그대로 복원된다.

---

## M9. 튜토리얼 + 학습 카드 정리

**목표**: 처음 하는 학생도 개념을 이해하며 시작할 수 있다.

- [ ] 첫 진입 튜토리얼(핵심 조작 + LTV·이자 안내)
- [ ] 학습 개념 모아보기(도감/카드) — `concepts.js` 기반
- [ ] 각 개념을 PDF 근거에 맞춰 초등·중1 눈높이로 최종 검수

**완료 기준**: 튜토리얼을 따라가면 한 턴을 스스로 진행할 수 있다.

---

## M10. 밸런싱 + 폴리싱

**목표**: 게임이 재미있게 굴러가고 보기 좋다.

- [ ] `balance.js` 수치 실제 플레이로 조정 (원진님과 상의)
- [ ] 파산/현금 부족 회복 루프(무한 모드용) 확정·구현
- [ ] 이벤트 카드 다양화
- [ ] 도트+부드러운 UI 시각 다듬기, 애니메이션·사운드(선택)
- [ ] 모바일 실제 기기 테스트

**완료 기준**: 여러 턴을 이어서 해도 밸런스가 무너지지 않고 학습 목표가 달성된다.

---

## 진행 원칙 요약

- **핵심 학습 마일스톤은 M3→M4→M5** (LTV → 이자·금리 → 시세차익). 여기가 이 게임의 교육적 뼈대.
- 각 ⚠️ 상의 포인트에서 멈추고 원진님과 수치를 확정한 뒤 넘어감.
- 결정이 바뀌면 `CLAUDE.md`의 해당 항목과 Changelog를 갱신.
